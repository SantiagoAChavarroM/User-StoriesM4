This project corresponds to **User Story Week 1 of Module 4 at Riwi**.

The main objective of this user story was to design a relational database for the hospital "Vida Sana", applying conceptual modeling (ERD) and logical modeling (Relational Model) following normalization principles up to Third Normal Form (3NF).

---

## Objective

Design a consistent and normalized relational database that manages:

- Patients
- Doctors
- Specialties
- Appointments
- Diagnoses

The system ensures:
- Data integrity
- Elimination of redundancy
- Proper relationships
- Normalization up to 3NF

---

## Concepts Applied

- Entity–Relationship Diagram (ERD)
- Relational Model transformation
- Primary Keys (PK)
- Foreign Keys (FK)
- 1:N relationships
- 1:1 relationships
- Database normalization (1NF, 2NF, 3NF)

---

## Deliverables

- `DER.pdf` → Conceptual Model (Entity–Relationship Diagram)
- `ModeloRelacional.pdf` → Logical Relational Model

---

## Learnings

During this project, the following concepts were reinforced:

- Conceptual database design
- Logical transformation of ERD into relational schema
- Relationship modeling (1:N and 1:1)
- Elimination of transitive dependencies
- Application of normalization rules up to 3NF
- Structural consistency between conceptual and logical models

---

## Final Result

A fully normalized relational database design, scalable and ready for SQL implementation.
