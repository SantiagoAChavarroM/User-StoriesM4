This project corresponds to **User Story Week 2 of Module 4 at Riwi**.

The main objective was to design and implement a complete academic database system using SQL, applying DDL, DML, DQL, DCL, and TCL operations.

---

## 🎯 Objective

Develop a fully functional relational database system to manage:

- Students  
- Teachers  
- Courses  
- Enrollments  

The system ensures:

- Data integrity through constraints  
- Analytical queries for academic performance  
- Subqueries and aggregations  
- View creation for structured reporting  
- Role-based access control  
- Transaction management  

---

## 🧩 Technologies Used

- MySQL 8.x  
- SQL (DDL, DML, DQL, DCL, TCL)  
- Visual Studio Code  

---

## 📁 Project Structure

gestion_academica.sql  

The script includes:

- Database creation  
- Table definitions with constraints (PK, FK, NOT NULL, UNIQUE, CHECK)  
- Data insertion  
- Analytical queries (JOIN, GROUP BY, HAVING)  
- Subqueries with IN and AVG  
- Aggregation functions (ROUND, SUM, MAX, MIN, COUNT)  
- View creation (`vista_historial_academico`)  
- Role management (GRANT / REVOKE)  
- Transaction handling (START TRANSACTION, SAVEPOINT, ROLLBACK, COMMIT)
